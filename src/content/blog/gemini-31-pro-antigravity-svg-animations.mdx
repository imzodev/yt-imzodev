# Gemini 3.1 PRO + Antigravity: Creating SVG Animations Made Super Easy

SVG animations have always been powerful but complex. Between path manipulation, timing functions, and browser compatibility, they can quickly become a headache. But what if you could generate stunning SVG animations with just natural language prompts?

Enter the dream team: **Gemini 3.1 PRO** + **Antigravity**.

## What is Gemini 3.1 PRO?

Gemini 3.1 PRO is Google's latest multimodal AI model that excels at understanding and generating code. Unlike previous versions, it has enhanced capabilities for:

- **Code generation** with context awareness
- **Visual reasoning** to understand SVG structures
- **Multi-step problem solving** for complex animations
- **Real-time iteration** and refinement

## What is Antigravity?

Antigravity is a lightweight JavaScript library that simplifies SVG animation. It provides:

- Declarative animation syntax
- Hardware-accelerated performance
- Built-in easing functions
- Cross-browser compatibility

## Why Combine Them?

The magic happens when you let Gemini handle the creative and logical heavy lifting, while Antigravity handles the technical implementation:

1. **Describe your vision** → Gemini generates the SVG paths and animation logic
2. **Copy the output** → Paste into Antigravity
3. **Tweak parameters** → Fine-tune timing and easing
4. **Export** → Production-ready SVG animation

```bash
npm install @antigravity/core

yarn add @antigravity/core
```

### 3. Use Gemini to Generate SVG Content

**Prompt to Gemini:**
```
Create an SVG of a bouncing ball that follows a physics-based trajectory. The ball should:
- Start at coordinates (100, 100)
- Move to (300, 100) in 1 second
- Bounce up to (300, 50) in 0.3 seconds
- Fall back to (300, 100) in 0.3 seconds
- Use easing: cubic-bezier(0.25, 0.1, 0.25, 1)
- Color: #3B82F6 (blue)
```

**Gemini generates:**

```javascript
const svgContent = `
  <svg width="500" height="200" viewBox="0 0 500 200">
    <circle id="bouncing-ball" 
            cx="100" cy="100" 
            r="20" 
            fill="#3B82F6" />
  </svg>
`;
```

### 4. Integrate with Antigravity

```javascript
import { animate } from '@antigravity/core';

// Container setup
const container = document.getElementById('animation-container');
container.innerHTML = svgContent;

// Animate the ball
const ball = document.getElementById('bouncing-ball');

animate({
  element: ball,
  properties: {
    cx: {
      from: 100,
      to: 300,
      duration: 1000,
      easing: 'cubic-bezier(0.25, 0.1, 0.25, 1)'
    },
    cy: [
      // Move horizontally
      {
        from: 100,
        to: 100,
        duration: 1000,
        easing: 'linear'
      },
      // Bounce up
      {
        from: 100,
        to: 50,
        duration: 300,
        easing: 'ease-out'
      },
      // Fall down
      {
        from: 50,
        to: 100,
        duration: 300,
        easing: 'ease-in'
      }
    ]
  }
});
```

## Advanced Example: Animated Icon

Let's create something more complex — an animated heart icon that beats.

### Gemini Prompt

```
Generate an SVG heart icon with a beating animation. The heart should:
- Have a modern, clean design
- Pulse rhythmically (like a heartbeat: lub-dub)
- Scale up 1.1x on first beat, then 1.15x on second
- Each beat cycle should last 1 second
- Use a gradient fill from #EF4444 to #DC2626
- Center in a 100x100 viewBox
```

### Generated Code

```javascript
const heartIcon = `
  <svg width="100" height="100" viewBox="0 0 100 100">
    <defs>
      <linearGradient id="heartGradient" x1="0%" y1="0%" x2="100%" y2="100%">
        <stop offset="0%" style="stop-color:#EF4444;stop-opacity:1" />
        <stop offset="100%" style="stop-color:#DC2626;stop-opacity:1" />
      </linearGradient>
    </defs>
    <path id="heart" 
          fill="url(#heartGradient)"
          d="M50 90
             C 30 70, 10 50, 10 35
             C 10 20, 25 10, 50 10
             C 75 10, 90 20, 90 35
             C 90 50, 70 70, 50 90Z" />
  </svg>
`;

animate({
  element: document.getElementById('heart'),
  properties: {
    transform: [
      // First beat (lub)
      {
        scale: { x: 1.1, y: 1.1 },
        duration: 150,
        easing: 'ease-out'
      },
      // Return to normal
      {
        scale: { x: 1, y: 1 },
        duration: 100,
        easing: 'ease-in'
      },
      // Second beat (dub) - stronger
      {
        scale: { x: 1.15, y: 1.15 },
        duration: 150,
        easing: 'ease-out'
      },
      // Return to normal
      {
        scale: { x: 1, y: 1 },
        duration: 600,
        easing: 'ease-in-out'
      }
    ],
    loop: true,
    loopDelay: 0
  }
});
```

## Pro Tips for Better Animations

### 1. Use GPU-Accelerated Properties

Antigravity automatically optimizes for GPU acceleration when possible. Focus on these properties:

- `transform` (translate, scale, rotate)
- `opacity`
- `filter`

**Avoid animating:** `width`, `height`, `top`, `left` — these trigger reflows.

### 2. Layer Multiple Animations

Complex animations are often a combination of simpler ones:

```javascript
// Animate multiple properties simultaneously
animate({
  element: document.getElementById('my-element'),
  properties: {
    translate: { x: 200, y: 100 },
    scale: { x: 1.5, y: 1.5 },
    rotate: 45,
    opacity: { from: 0, to: 1 },
    duration: 800,
    easing: 'cubic-bezier(0.34, 1.56, 0.64, 1)' // Back easing
  }
});
```

### 3. Use Delays for Sequences

```javascript
// Create a sequence: move, then fade, then scale
animate({
  element: element1,
  properties: { translate: { x: 100 } },
  duration: 500,
  delay: 0
});

animate({
  element: element1,
  properties: { opacity: { from: 0, to: 1 } },
  duration: 300,
  delay: 500  // Starts after first animation
});

animate({
  element: element1,
  properties: { scale: { x: 1.2, y: 1.2 } },
  duration: 400,
  delay: 800  // Starts after second animation
});
```

### 4. Easing Functions Reference

| Easing | Effect | Best For |
|--------|--------|----------|
| `linear` | Constant speed | Unusual, specific needs |
| `ease` | Gentle start/end | General purpose |
| `ease-in` | Slow start | Entering elements |
| `ease-out` | Slow end | Leaving elements |
| `ease-in-out` | Slow start/end | Natural motion |
| `cubic-bezier(...)` | Custom | Precise control |

## Troubleshooting

### Animation not smooth?

1. Check for conflicting CSS transitions
2. Use `will-change` property for animated elements
3. Reduce number of animated elements
4. Test on lower-end devices

### SVG looks blurry?

1. Use proper `viewBox` settings
2. Avoid scaling beyond 2x
3. Check `shape-rendering` CSS property
4. Ensure crisp coordinate values (avoid decimals)

### Performance issues?

1. Use `requestAnimationFrame` (Antigravity does this by default)
2. Limit concurrent animations
3. Use CSS `transform` instead of position changes
4. Consider using `content-visibility: auto` for off-screen elements
